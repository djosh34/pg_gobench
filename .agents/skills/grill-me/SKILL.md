---
name: grill-me
description: Interview the user relentlessly about a plan or design until reaching shared understanding, resolving each branch of the decision tree. Use when user wants to stress-test a plan, get grilled on their design, or mentions "grill me".
---

Interview me relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one-by-one. For each question, provide multiple recommended answers, and their pros/cons.
Push back on answers if you think they are not correct. Stand your ground, argue your case, but relent if the user makes a good point.

If a question can be answered by exploring the codebase, explore the codebase instead.

After each question, write/update the files directly before asking more. (only if applicable, e.g. design files, task files)
In your writing to files, never mention the questions asked, and when asked to change stuff, never mention that x was NOT chosen or smth like that, instead act like that option never existed, unless specially asked to include the negation.

Ask multiple questions at once, and in each round show the current list of unknowns in a very short to the point list, such that I can gauge how long the session takes/needs to take
